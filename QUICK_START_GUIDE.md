# JIRA工单效率分析系统 - 快速启动指南

## 🎯 系统概述

JIRA工单效率分析系统（JTAS）是一个专业的Web应用，用于分析和对比AI自动分单与传统分单的效率差异。系统支持20多位处理人员的效率分析，提供实时仪表板、深度对比分析和数据导入功能。

## 🚀 快速启动

### 方法一：Docker部署（推荐）

```bash
# 1. 克隆项目
git clone <repository-url>
cd JTAS

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，设置数据库密码等配置

# 3. 启动所有服务
docker-compose up -d

# 4. 等待服务启动完成
docker-compose ps

# 5. 访问应用
# 前端界面: http://localhost:3000
# API文档: http://localhost:8080/api/v1/swagger-ui.html
# 监控面板: http://localhost:3001 (用户名: admin, 密码: admin123)
```

### 方法二：本地开发

```bash
# 后端启动
cd JTAS
mvn clean package -DskipTests
java -jar target/*.jar

# 前端启动（新终端）
cd frontend
npm install
npm start
```

## 📊 核心功能特性

### 1. 实时效率仪表板
- **关键指标展示**：总工单数、AI分单占比、平均处理时间、完成率
- **趋势分析**：AI vs 传统分单效率对比趋势图
- **部门分布**：各部门工单分布和处理情况
- **个人排名**：处理人员效率排行榜

### 2. 深度对比分析
- **统计分析**：平均处理时间、响应时间、完成率、客户满意度
- **显著性检验**：统计学显著性验证分析结果可靠性
- **改进度量化**：量化AI分单相对于传统分单的改进百分比
- **报告导出**：支持PDF和Excel格式的专业分析报告

### 3. 智能数据导入
- **多格式支持**：JSON、CSV、Excel格式的JIRA数据导入
- **数据验证**：自动验证数据完整性和格式正确性
- **实时进度**：导入过程实时进度监控
- **批量处理**：支持多文件批量导入

### 4. 瓶颈识别与优化
- **自动识别**：处理时间过长、工作负载过高的瓶颈点
- **工作负载均衡分析**：各处理人员工作负载分布
- **效率优化建议**：基于数据分析的优化建议

## 💾 数据模型

### 核心数据表
- **ticket_record**: 工单记录表，存储JIRA工单详细信息
- **assignee_info**: 处理人员信息表，存储20多位同事的基本信息和技能
- **efficiency_metrics**: 效率指标表，存储计算后的效率指标数据
- **comparison_result**: 对比分析结果表，存储AI vs 传统分单的对比结果
- **import_log**: 导入日志表，记录数据导入的历史和状态

## 🔍 关键指标说明

### AI分单 vs 传统分单对比指标
- **平均处理时间**：从分配到解决的平均时间
- **平均响应时间**：从分配到首次响应的平均时间  
- **完成率**：在指定时间内完成的工单比例
- **客户满意度**：基于处理质量的满意度评分

### 效率改进计算公式
```
处理时间改进率 = (传统分单平均时间 - AI分单平均时间) / 传统分单平均时间 × 100%
```

## 🚨 故障排查

### 常见问题及解决方案

1. **数据库连接失败**
   ```bash
   # 检查数据库服务状态
   docker-compose logs postgres
   
   # 重启数据库服务
   docker-compose restart postgres
   ```

2. **前端无法访问后端API**
   ```bash
   # 检查后端服务状态
   curl http://localhost:8080/api/v1/actuator/health
   
   # 查看后端日志
   docker-compose logs jtas-backend
   ```

3. **导入数据验证失败**
   - 检查数据格式是否符合模板要求
   - 确认必填字段是否完整
   - 使用提供的Python验证脚本预检查
